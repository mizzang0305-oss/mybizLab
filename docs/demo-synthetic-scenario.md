# Demo Synthetic Scenario

This document defines fictional demo data for the July 2026 rehearsal package.

The scenario is synthetic-only. It must not be replaced with real customer names, raw phone numbers, raw email addresses, real order rows, real reservations, real waiting entries, real store records, or production customer samples.

## Scenario purpose

The owner should understand the MyBiz value path in three minutes:

- customer memory makes repeat visits easier to recognize
- VIP customer candidates make revisit and average-order-value opportunities visible
- campaign preparation preview creates a message draft without sending it
- Growth 99,000 KRW is the recommended pilot offer for the memory-based revenue engine

## Fictional store context

- store label: `synthetic_pilot_store`
- store type: cafe or dessert shop
- pilot mode: read-only revenue proof
- actual store creation: disabled
- actual customer import: disabled

## Privacy and consent boundary

- no real customer data is read
- no raw phone or email is resolved
- no recipient list is exported
- no marketing consent is written
- no provider API is called
- no SMS/Kakao/Email delivery is available
- no payment or subscription is created

## Rehearsal checklist

- explain the problem in 1 minute
- explain MyBiz screen value in 1 minute
- pitch Growth 99,000 KRW in 1 minute
- state that all data is synthetic
- state that actual delivery and payment are disabled
- state that privacy and marketing consent require separate approval
- avoid revenue guarantee wording
