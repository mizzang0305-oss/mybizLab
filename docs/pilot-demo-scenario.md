# Pilot Demo Scenario

This document defines the July pilot demo scenario only.

It is read-only and plan-only. It does not log in to production as a real store owner, inspect real customer rows, create customers, send messages, charge payments, or connect providers.

## 1. 사장님 현재 문제 확인

- 단골 기억이 직원 머릿속에 있음
- 주문 취향이 시스템에 남지 않음
- 재방문 유도가 감으로 이루어짐
- 직원이 바뀌면 고객 기억이 사라짐
- 사장님은 "이번 주 누구를 다시 부를지" 바로 보기 어려움

## 2. MyBiz 가치 제시

- 고객 기억 카드
- VIP 고객 후보
- 취향/방문/주문 맥락 preview
- 다음 주문/재방문 준비 preview
- 캠페인 준비 미리보기
- 확인 전용 리포트

## 3. 돈 받을 포인트

- 단골을 놓치지 않음
- 추천과 재주문 준비가 쉬워짐
- 직원이 바뀌어도 고객 기억이 유지됨
- 실제 발송 전에도 사장님이 "누구에게 무엇을 제안할지" 볼 수 있음
- Growth 99,000원 플랜의 핵심 가치를 기억 기반 매출 엔진으로 설명할 수 있음

## 4. 안전 경계 설명

- 실제 발송 없음
- raw phone/email 접근 없음
- 개인정보/마케팅 동의 별도 필요
- 결제 자동화 없음
- 파일럿은 read-only revenue proof
- 실제 파일럿 매장 생성은 별도 Owner 승인 후 진행
- 실제 고객 데이터 import는 별도 개인정보/동의 검토 후 진행

## Demo Close

The close should ask whether the owner wants to join the read-only July pilot.

Do not promise actual SMS/Kakao/Email delivery, automatic customer grade changes, payment automation, or live customer import during this step.

Before a live owner conversation, rehearse the 3-minute synthetic-only flow in `docs/demo-rehearsal-script.md` and `docs/demo-synthetic-scenario.md`. The rehearsal package uses fictional customer memory examples only and must not include real customer data.
