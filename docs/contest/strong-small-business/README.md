# MyBiz x 강한소상공인 공모전 문서 묶음

이 폴더는 `MyBiz`를 `강한 소상공인 성장지원사업` 또는 유사한 공모형 선발 프로그램에 제출할 때 바로 가져다 쓸 수 있도록 만든 한국어 초안 묶음입니다.

## 이 폴더에 들어있는 파일

- `application-draft-ko.md`
  - 실제 신청서 문항에 옮겨 적기 좋은 장문/중문 답변 초안
- `product-evidence-ko.md`
  - 현재 저장소와 GitHub 흐름을 바탕으로 정리한 제품 근거
- `judging-fit-ko.md`
  - 공식 공고 기준과 MyBiz의 적합도, 전략, 리스크
- `pitch-outline-ko.md`
  - 발표/피칭용 3분, 5분 구조
- `submission-checklist-ko.md`
  - 제출 전에 실제로 채워 넣어야 할 자료와 체크리스트
- `mybiz-contest-facts.json`
  - 재활용 가능한 구조화 팩트 시트

## 가장 중요한 결론

MyBiz의 가장 강한 한 줄은 아래입니다.

> MyBiz는 공개 스토어, 문의, 예약, 웨이팅, 주문, 설문에서 들어오는 고객 신호를 하나의 고객 기억으로 묶고, AI가 다음 액션과 운영 리포트로 바꿔 반복 매출을 만드는 고객기억 기반 매출 시스템입니다.

이 프로젝트를 공모전에 낼 때는 `홈페이지 제작 도구`, `챗봇`, `마케팅 툴`로 보이게 하면 약해집니다.  
반드시 `소상공인의 흩어진 고객 신호를 기억 자산으로 전환하는 운영 시스템`으로 설명해야 합니다.

## 추천 제출 전략

가장 추천하는 전략은 아래 순서입니다.

1. `MyBiz`를 순수 SaaS처럼 소개하지 말고, 실제 매장 운영 문제를 해결하는 `운영서비스`로 설명합니다.
2. `공개 페이지 -> 문의/예약/웨이팅 유입 -> 고객 기억 결합 -> AI 실행안 -> 대시보드` 흐름을 하나의 루프로 보여줍니다.
3. 반드시 `대표 매장 1곳` 또는 `대표 업종 1개`를 전면에 세워 실증형 사례처럼 보이게 만듭니다.
4. 숫자는 적게 써도 되지만, `전/후 변화`는 꼭 넣습니다.
5. `고객 기억`을 핵심 차별점으로 반복해서 강조합니다.

## 트랙 적합성 메모

지금 확인한 공식 자료 기준으로 `강한 소상공인 성장지원사업`은 2025년에 아래 유형을 운영했습니다.

- 트랙 I: 라이프스타일, 로컬브랜드, 장수소상공인
- 트랙 II: 온라인 셀러
- 트랙 III: 글로벌

현재 MyBiz는 `순수 B2B SaaS`로만 제출하면 트랙 적합성이 애매할 수 있습니다.  
가장 안전한 방식은 `실제 운영 중인 대표 소상공인 브랜드/매장`을 중심에 두고, MyBiz를 그 브랜드의 차별화된 운영서비스이자 성장 엔진으로 설명하는 것입니다.

이 판단 근거와 대응 전략은 `judging-fit-ko.md`에 자세히 정리했습니다.

## 2026 기준 해석 메모

2026년 4월 19일 기준, 제가 확인한 공식 자료에서는 `2026년 강한 소상공인 성장지원사업`의 전용 모집공고 본문까지는 확인하지 못했습니다.  
대신 `2026년 소상공인 지원사업·융자 통합 공고`와 관련 보도자료는 확인됐고, 여기서 `AI·디지털 전환 중심` 지원 확대 기조가 드러납니다.

따라서 이 문서 묶음은 아래 원칙으로 작성했습니다.

- 세부 트랙/선발 절차/금액은 `가장 최근에 상세 확인 가능한 2025 공식 공고` 기준으로 정리
- 2026년은 `정책 기조와 통합 공고 존재`를 참고사항으로만 반영
- 최종 제출 전에는 반드시 `sbiz24`와 `mss.go.kr`의 최신 모집공고를 다시 대조

## 사용 순서

1. `judging-fit-ko.md`를 먼저 읽고 어떤 트랙/포지셔닝으로 갈지 결정합니다.
2. `application-draft-ko.md`에서 문항별 문장을 골라 실제 신청서에 넣습니다.
3. `product-evidence-ko.md`를 참고해 스크린샷과 수치를 붙입니다.
4. `submission-checklist-ko.md`를 보면서 빠진 실증 자료를 채웁니다.
5. 발표가 있다면 `pitch-outline-ko.md`로 3분 버전을 먼저 맞춥니다.

## 꼭 직접 채워 넣어야 하는 항목

아래는 저장소만으로는 확정할 수 없어서 반드시 실데이터로 교체해야 합니다.

- 실제 대표 매장명
- 실제 대표자 소개

- 실제 재방문 또는 매출 변화
- 실제 고객 후기
- 실제 도입 전/후 운영 변화
