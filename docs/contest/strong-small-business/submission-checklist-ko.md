# MyBiz 공모전 제출 체크리스트

## 1. 신청 전략 체크

- [ ] 어떤 트랙으로 제출할지 결정했다
- [ ] 대표 매장 또는 대표 업종 사례를 정했다
- [ ] MyBiz를 `SaaS`가 아니라 `소상공인 운영서비스`로 설명한다
- [ ] 핵심 문장을 `고객 기억 기반 반복 매출 구조`로 통일했다

## 2. 숫자 체크

아래는 반드시 실제 수치로 채워야 합니다.

- [ ] 월 공개 유입 수
- [ ] 문의 수
- [ ] 예약 수
- [ ] 웨이팅 수
- [ ] 고객 누적 수
- [ ] 재방문율 또는 재구매율
- [ ] 응대 시간 단축 수치
- [ ] 매출 또는 전환 개선 수치

## 3. 대표 사례 체크

- [ ] 대표 매장명
- [ ] 업종
- [ ] 지역
- [ ] 도입 전 문제
- [ ] 도입 후 변화
- [ ] 실제 점주 코멘트 1개 이상

## 4. 화면 자료 체크

아래 캡처를 준비하면 좋습니다.

- [ ] 공개 스토어 첫 화면
- [ ] 문의 화면
- [ ] 예약 화면
- [ ] 웨이팅 화면
- [ ] 고객 관리 화면
- [ ] 고객 타임라인 또는 메모리 관련 화면
- [ ] AI 매니저 또는 AI 리포트 화면
- [ ] 온보딩/진단 흐름
- [ ] 요금제 또는 스토어 활성화 흐름

## 5. 영상 자료 체크

- [ ] 30~60초 제품 데모 영상
- [ ] 공개 유입 -> 문의/예약/웨이팅 -> 고객 관리 -> AI 리포트 순서로 흐름이 보인다
- [ ] 실제 운영 맥락이 드러난다
- [ ] 화려한 애니메이션보다 문제 해결 흐름이 우선이다

## 6. 신청서 문장 체크

- [ ] “AI 챗봇”이라는 표현을 전면에 두지 않았다
- [ ] “홈페이지 제작”이라는 표현을 전면에 두지 않았다
- [ ] “고객 기억”, “반복 매출”, “운영서비스”를 핵심 키워드로 사용했다
- [ ] 문제 -> 구조 -> 효과 순서로 정리했다

## 7. 심사 대응 체크

- [ ] 차별성: 왜 기존 홈페이지/CRM/예약툴과 다른지 설명 가능
- [ ] 시장성: 어떤 업종에 확장 가능한지 설명 가능
- [ ] 실행력: 현재 동작형 제품이라는 근거가 있음
- [ ] 성장성: 무료 진입 -> 유료 확장 구조가 설명됨
- [ ] 팀 역량: 왜 이 팀이 이 문제를 풀 수 있는지 설명 가능

## 8. 최종 제출 전 확인

- [ ] 2026 최신 공고를 다시 확인했다
- [ ] 신청 자격 요건을 다시 확인했다
- [ ] 제출 파일명과 형식을 확인했다
- [ ] 대표자명, 사업자 정보, 연락처를 최신으로 확인했다
- [ ] 실제 운영 수치와 문서 수치가 일치한다

## 9. 심사위원 시점 최종 점검 질문

아래 질문에 한 문장으로 답할 수 있어야 합니다.

### 이 서비스는 정확히 무엇인가

- [ ] 답변 가능

### 왜 꼭 필요한가

- [ ] 답변 가능

### 기존 방식과 무엇이 다른가

- [ ] 답변 가능

### 실제로 이미 어디까지 만들어졌는가

- [ ] 답변 가능

### 소상공인이 이걸 쓰면 무엇이 달라지는가

- [ ] 답변 가능

## 10. 가장 추천하는 최종 한 문장

MyBiz는 소상공인의 문의, 예약, 웨이팅, 주문, 설문을 고객 기억 자산으로 전환해 반복 매출을 만드는 운영서비스입니다.
