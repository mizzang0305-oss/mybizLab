# Content SEO Launch Plan

This document defines the July 2026 content SEO launch plan only.

It does not publish blog posts, call external blog APIs, add OAuth clients, add API keys, add env vars, publish social posts, send messages, charge payments, read real customer data, or write customer data.

## Positioning

MyBiz is a memory-based revenue engine SaaS.

The July content narrative should make one problem obvious: stores without memory lose sales. When regular-customer preferences, visit timing, menu choices, and inquiry history are scattered across staff memory or order apps, the owner cannot consistently prepare the next visit or lift average order value.

## Solution Story

The safe July story is:

- customer memory card for masked, read-only customer context
- VIP candidate view for store_id-scoped opportunity discovery
- campaign prep preview that shows what the owner could suggest before any delivery exists
- delivery approval gate, consent model, raw recipient boundary, and audit log plan before future execution

This content should show what to propose to regular customers without real delivery. It must not claim actual SMS/Kakao/Email delivery, raw contact resolution, payment automation, or production data writes.

## Pilot Offer

The pilot CTA should use:

- Plan: Growth
- Pilot price: Growth 99,000 KRW
- CTA: pilot consultation request
- Safety note: no actual delivery, no raw contact, no payment automation

The CTA should route to a manual consultation path. It must not create leads, write customer records, charge payments, send messages, or publish to external channels automatically.

## Article Framing

Recommended article structure:

1. Problem: stores without memory lose sales.
2. Cost: missed revisit timing and weak average-order-value suggestions.
3. MyBiz angle: customer memory card and VIP candidate preview.
4. Trust boundary: preview-only campaign planning, no send, no raw contact, no payment automation.
5. Pilot offer: Growth 99,000 KRW.
6. CTA: pilot consultation request.

## Manual Publication Checklist

Before a manual post goes live:

- SEO title is written for the exact store-owner problem.
- meta description explains the read-only pilot value.
- canonical URL is selected.
- sitemap/RSS inclusion is planned for the selected publishing surface.
- internal link points to pricing, VIP Customer Memory, and pilot consultation request.
- UTM naming is defined for manual links.
- author/review flow is complete.
- no real customer case is included without separate approval.
- no raw phone, raw email, or customer row evidence is included.
